# 系统设计

## 架构图

```
React + ECharts → FastAPI → 5 Agents (共享内存)
                → ChromaDB (向量存储)
                → MiMo API (LLM)
```

## 技术选型

| 层 | 选择 | 理由 |
|---|---|---|
| 前端 | React 19 + shadcn/ui + ECharts | 交互丰富，可视化强 |
| 后端 | FastAPI | 异步高性能，自动 API 文档 |
| PDF 解析 | PyMuPDF + pymupdf4llm | 真文字 PDF，逐页解析 |
| Embedding | bge-small-zh-v1.5 | 本地运行，中文优化 |
| 向量库 | ChromaDB | 轻量级，持久化存储 |
| LLM | MiMo 2.5 Pro | OpenAI 兼容接口 |

## API 设计

| 端点 | 方法 | 功能 |
|---|---|---|
| /api/textbooks/upload | POST | 上传教材 |
| /api/textbooks/list | GET | 教材列表 |
| /api/knowledge/extract/{id} | POST | 知识提取 |
| /api/knowledge/graph/{id} | GET | 获取图谱 |
| /api/integration/merge | POST | 执行整合 |
| /api/integration/decisions | GET | 决策列表 |
| /api/rag/query | POST | RAG 问答 |
| /api/chat/send | POST | 对话交互 |

## 数据流

1. PDF → ParserAgent → TextbookInfo
2. TextbookInfo → ExtractorAgent → KnowledgeGraph
3. KnowledgeGraph → RAGAgent → ChromaDB
4. 多个 KnowledgeGraph → IntegratorAgent → MergedGraph
5. 前端展示图谱 + 决策 + RAG + 对话
