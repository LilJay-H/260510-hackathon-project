# 系统设计

## 架构图

```
React + ECharts → FastAPI → 5 Agents (共享内存)
                → ChromaDB (向量存储)
                → MiMo API (LLM)
```

## 技术选型

| 层 | 选择 | 理由 |
|---|---|---|
| 前端 | React 19 + shadcn/ui + ECharts | 交互丰富，可视化强 |
| 后端 | FastAPI | 异步高性能，自动 API 文档 |
| PDF 解析 | PyMuPDF + pymupdf4llm | 真文字 PDF，逐页解析 |
| Embedding | bge-small-zh-v1.5 | 本地运行，中文优化 |
| 向量库 | ChromaDB | 轻量级，持久化存储 |
| LLM | MiMo 2.5 Pro | OpenAI 兼容接口 |

## API 设计

### 教材管理

#### POST /api/textbooks/upload — 上传教材

**请求**: `multipart/form-data`
```
file: (PDF/MD/TXT 文件)
```

**成功响应** (200):
```json
{
  "id": "book_07",
  "title": "07_病理生理学",
  "chapters": 21,
  "total_chars": 381483
}
```

#### GET /api/textbooks/list — 教材列表

**响应** (200):
```json
[
  {"id": "book_07", "title": "07_病理生理学", "chapters": 21, "total_chars": 381483}
]
```

### 知识提取

#### POST /api/knowledge/extract/{textbook_id} — 执行知识提取

**路径参数**: `textbook_id` — 教材 ID

**响应** (200):
```json
{
  "textbook_id": "book_07",
  "nodes": 214,
  "edges": 235
}
```

### 跨教材整合

**响应** (200):
```json
{
  "original_chars": 3100000,
  "merged_chars": 890000,
  "compression_ratio": 0.287,
  "decisions_count": 399
}
```

**响应** (200):

  {

### RAG 问答

#### POST /api/rag/query — 提问

**请求体**:
```json
{
  "question": "什么是炎症？其基本病理变化有哪些？"
}
```

**响应** (200):
```json
{
  "answer": "炎症是具有血管系统的活体组织对损伤因子的防御性反应...",
  "citations": [
    {"textbook": "病理学", "chapter": "第四章 炎症", "page": 78, "score": 0.92},
    {"textbook": "病理生理学", "chapter": "第六章 炎症反应", "page": 112, "score": 0.85}
  ]
}
```

### 多轮对话

#### POST /api/chat/send — 发送消息

**请求体**:
```json
{
  "message": "为什么把炎症和免疫合并了？"
}
```

**响应** (200):
```json
{
  "response": "关于「炎症」的合并决策：三本教材对炎症的定义一致，保留《病理学》版本因其包含完整的病因-机制-变化-结局描述（置信度: 95%）"
}
```

## 数据流

1. PDF → ParserAgent → TextbookInfo
2. TextbookInfo → ExtractorAgent → KnowledgeGraph
3. KnowledgeGraph → RAGAgent → ChromaDB
4. 多个 KnowledgeGraph → IntegratorAgent → MergedGraph
5. 前端展示图谱 + 决策 + RAG + 对话
