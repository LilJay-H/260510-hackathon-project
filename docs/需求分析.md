# 需求分析

## 子问题分解

| 子问题 | 对应 Agent | 输入 | 输出 |
|---|---|---|---|
| 多格式教材解析 | ParserAgent | PDF 文件 | 结构化章节数据 |
| 知识点提取 | ExtractorAgent | 章节内容 | 知识图谱（节点+边） |
| 跨教材去重 | IntegratorAgent | 多个知识图谱 | 合并后图谱+决策 |
| 精准问答 | RAGAgent | 用户问题 | 带引用的回答 |
| 对话修改方案 | ChatAgent | 用户指令 | 修改后的图谱 |

## 知识点粒度定义

**原则**: 每个知识点是一个独立的、可被查询的知识单元。

**标准**:
- 一个知识点对应一个可独立回答的问题
- 不将多个无关概念合并为一个节点
- 不将过于细碎的信息单独建节点

**分类**: 核心概念、解剖结构、生理过程、病理变化、临床表现、诊断方法、治疗原则

## 重复判定标准

| 判定为重复 | 判定为不同 |
|---|---|
| 同一概念不同措辞描述 | 同一名称不同学科语境含义不同 |
| 上位/下位概念重复定义 | 描述角度不同但互补 |

## 压缩比目标

- 原始: 7 本 × ~370 页 × ~1200 字/页 ≈ 3,100,000 字
- 目标: 整合后 ≤ 930,000 字（30%）
- 策略: merge 合并重复，remove 删除冗余
